# الأثير

**الأثير** — منصة ويب لعرض مسارات جمعية المشي والجري بالأحساء (رياضية، فلكية، ثقافية، وغيرها) مع تفاصيل كل مسار وروابط الحجز عند توفرها.

---

## 1. نبذة عن فكرة المشروع

يهدف **الأثير** إلى تجميع معلومات المسارات المعلن عنها على موقع جمعية المشي والجري بالأحساء في واجهة عربية واحدة، سهلة القراءة، ومناسبة للهواتف وأجهزة سطح المكتب. المشروع موجّه لمن يرغب في الاطلاع على البرامج، ومقارنة المسارات، والانتقال السريع إلى صفحة الحجز الرسمية عند توفر الرابط.

---

## 2. التقنيات المستخدمة

| التقنية | الدور |
|--------|--------|
| **HTML5** | هيكلة الصفحات، العناوين، والروابط مع `dir="rtl"` و`lang="ar"`. |
| **CSS3** | تنسيق الواجهة، الشبكة المتجاوبة، الألوان، والظلال دون أي إطار عمل خارجي. |
| **JavaScript (Vanilla)** | ملفات خفيفة للتوسعة لاحقاً؛ لا اعتماد على مكتبات. |
| **PHP 8+** | توليد الصفحات، الاتصال بقاعدة البيانات، لوحة التحكم، وواجهة جلب البيانات. |
| **MySQL** | تخزين المسارات والإعدادات، مع تمييز المسار بـ `source_url` لمنع التكرار. |

---

## 3. هيكل قاعدة البيانات

**اسم القاعدة:** `atheer_db` (ترميز `utf8mb4`).

### جدول `routes`

يخزن كل مسار معروضاً في الموقع.

| الحقل | الوصف |
|--------|--------|
| `id` | مفتاح رقمي تلقائي. |
| `name` | اسم المسار. |
| `description` | وصف نصي. |
| `date` | تاريخ الفعالية (إن وُجد). |
| `time` | وقت الانطلاق بتنسيق وقت قاعدة البيانات. |
| `meeting_point` | نقطة التجمع. |
| `price` | الرسوم بالريال. |
| `min_participants` / `max_participants` | الحد الأدنى والأقصى للمشاركين عند توفرهما في المصدر. |
| `location` | منطقة جغرافية أو وصف موقعي. |
| `itinerary` | نص JSON لخريطة يومية أو فقرات البرنامج. |
| `image_url` | رابط صورة. |
| `source_url` | رابط الصفحة الأصلية على موقع الجمعية (مفتاح منطقي للتحديث). |
| `booking_url` | رابط الحجز أو الدفع عند توفره. |
| `type` | نوع المسار: رياضي، فلكي، ثقافي، زراعي، مهاجر. |
| `is_active` | إظهار المسار في الواجهة العامة أو إخفاؤه. |
| `created_at` / `updated_at` | طوابع زمنية. |

### جدول `settings`

أزواج `key_name` / `key_value` لإعدادات عامة (مثل عنوان الموقع في قاعدة البيانات).

## 4. الخوارزميات والمنطق

### جلب البيانات من مواقع WRA (استخراج من HTML)

1. لكل رابط من قائمة المسارات السبعة يتم طلب الصفحة عبر **cURL**، وعند تعذّر ذلك يُستخدم **`file_get_contents`** مع سياق مناسب.
2. يُحمّل HTML في **`DOMDocument`** ويُحلَّل بـ **`DOMXPath`**:
   - العنوان من أول **`h1`** أو من **`og:title`**.
   - الوصف من فقرة طويلة أو من نص الصفحة بعد تنظيف المسافات.
   - التاريخ بأنماط عربية شائعة (يوم + شهر عربي + سنة).
   - الوقت من عبارات مثل «عصر» و«صباحاً».
   - الرسوم من كلمة «ريال» أو سياق «رسوم».
   - نقطة التجمع من عبارة «نقطة التجمع».
   - الصورة من **`og:image`** أو أول صورة مناسبة في الصفحة.
   - رابط الحجز من روابط تحتوي `checkout` أو نص «احجز».
3. يُبنى حقل **`itinerary`** كـ JSON عند وجود أسطر مرقّمة أو فقرات تبدأ بعلامات تعداد.
4. يُستعلَم عن **`source_url`**: إن وُجد صف **UPDATE**، وإلا **INSERT**.

### عرض المسارات والتصفح

- الصفحة الرئيسية **`index.php`** تستدعي **`getAllRoutes()`** لعرض الشبكة.
- **`route.php?id=`** يستدعي **`getRouteById()`** ويعرض التفاصيل ويفكّك JSON للمسار اليومي عند الحاجة.

### التخزين والتحديث

- المصدر الرسمي للتمييز هو **`source_url`**؛ المحتوى الغني يُدمَج من **`includes/route_seed_content.php`** تلقائياً عند الحاجة (علم في **`settings`**).
- التعديل اليدوي متاح من **`admin/update_routes.php`** لضبط أي حقل.

## 6. تدفق البيانات من البداية للنهاية

1. يُنشأ مخطط قاعدة البيانات من **`database/atheer.sql`**.
2. يُنسخ **`config/database.example.php`** إلى **`config/database.local.php`** وتُعدَّل بيانات الاتصال (هذا الملف لا يُرفع إلى Git).
3. يُرفع المجلد إلى الخادم المحلي (مثل `htdocs/AL-Atheer`).
4. تُملأ **`routes`** من **`database/atheer.sql`**؛ عند أول تشغيل للموقع يُكمَّل المحتوى المرجعي من الكود إن لزم.
5. يزور الزائر **`index.php`** فيقرأ الصفوف النشطة من **`routes`**، ثم ينتقل إلى **`route.php`** أو رابط الحجز الخارجي.

## 7. مقاطع كود مهمة (مفاهيم)

### دالة جلب المسارات للعرض

في **`includes/functions.php`** تُعاد كل المسارات النشطة مرتبة:

```php
function getAllRoutes(): array
{
    $pdo = get_pdo();
    $sql = 'SELECT * FROM routes WHERE is_active = 1 ORDER BY `date` IS NULL, `date` ASC, `name` ASC';
    $stmt = $pdo->query($sql);
    return $stmt->fetchAll();
}
```

### استعلام التحديث عند وجود المصدر

في **`api/fetch_routes.php`** يُستخدم نفس المعاملات للتحديث حسب `source_url` (مختصر المفهوم):

```php
$updateSql = 'UPDATE routes SET
    name = :name, description = :description, date = :date, time = :time, meeting_point = :meeting_point,
    price = :price, min_participants = :minp, max_participants = :maxp, location = :location,
    itinerary = :itinerary, image_url = :image_url, booking_url = :booking_url, type = :type
    WHERE source_url = :source_url';
```

### عرض مسار بالمعرف

```php
function getRouteById(int $id): ?array
{
    if ($id < 1) {
        return null;
    }
    $pdo = get_pdo();
    $stmt = $pdo->prepare('SELECT * FROM routes WHERE id = :id AND is_active = 1 LIMIT 1');
    $stmt->execute(['id' => $id]);
    $row = $stmt->fetch();
    return $row ?: null;
}
```

## 8. خطوات التثبيت والتشغيل

### أ. المتطلبات

- PHP 8.0 أو أحدث مع امتدادات **PDO MySQL** و**dom** و**libxml** و**curl** (مستحسن).
- MySQL 5.7+ أو MariaDB متوافق.
- Apache أو Nginx مع دعم PHP.

### ب. إنشاء القاعدة

نفّذ محتوى **`database/atheer.sql`** من أداة مثل phpMyAdmin أو سطر أوامر MySQL لإنشاء **`atheer_db`** والجداول.

### ج. الإعداد

1. انسخ **`config/database.example.php`** إلى **`config/database.local.php`** وعدّل: `DB_HOST`، `DB_USER`، `DB_PASS`، و`DB_NAME`.
2. انسخ **`config/auth.example.php`** إلى **`config/auth.local.php`** واختر اسم مستخدم وكلمة مرور قوية للمدير.

ملفات **`.local.php`** تبقى على جهازك أو على الاستضافة فقط ولا تُرفع إلى GitHub.

### د. النشر على الاستضافة (مثل Hostinger)

1. ارفع ملفات المشروع عبر مدير الملفات أو FTP.
2. أنشئ قاعدة بيانات MySQL من لوحة التحكم ونفّذ **`database/atheer.sql`**.
3. أنشئ يدوياً على السيرفر **`config/database.local.php`** و**`config/auth.local.php`** ببيانات الاستضافة (لا تضعها في المستودع).

### هـ. النشر المحلي

ضع المجلد **`AL-Atheer`** تحت جذر الويب (مثال: `C:\xampp\htdocs\AL-Atheer`).

### و. المحتوى والتعديل

بعد تسجيل الدخول إلى لوحة التحكم يمكنك تعديل أي مسار من **تعديل**. لا يُعرض زر جلب تلقائي من المواقع الخارجية حتى لا يُستبدل المحتوى المدروس دون قصد.

### ز. التصفح

افتح المتصفح على عنوان مثل:

`http://localhost/AL-Atheer/`

## 10. ملاحظات للمطورين

- ترميز الملفات **UTF-8**؛ الواجهة **RTL** بالكامل.
- هيكل HTML للمواقع الخارجية قد يتغيّر؛ عند اختلال جزء من الاستخراج يُفضّل التصحيح يدوياً من لوحة التحكم.
- حماية مجلد **`admin`** إضافياً على مستوى الخادم (مثلاً تقييد IP) يزيد الأمان في بيئة الإنتاج.
- استبدل **`assets/images/logo.png`** بشعار الجمعية الفعلي بنفس الاسم أو حدّث المسار في **`includes/header.php`**.
